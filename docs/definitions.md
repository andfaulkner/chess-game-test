Client-server model
===================
Really straightforward:

.--------.                .--------.
|        |---[request]--->|        |
| Client |                | Server |
|        |<--[response]---|        |
'--------'                '--------'

-   Server contains handlers that:
    -   "Catch" requests on different routes
    -   Process them
    -   Return a result based on the processing

GET requests:    Simple static requests for data (no attached data, URL-only)
POST requests:   Requests to add data
PUT requests:    Requests to modify data
DELETE requests: Requests to remove data

All web applications work this way
